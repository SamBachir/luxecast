# Anti-Bypass Protection

## Core Rules
1. Contact details stay hidden before confirmed booking
2. All payments must happen inside platform
3. Repeat bookings within lock period must remain on-platform
4. Suspicious chat patterns trigger risk scoring
5. Confirmed bypass triggers penalties, restrictions, or bans

## Detection Signals
- phone numbers or emails in chat
- phrases like 'call me', 'off app', 'cheaper direct'
- first booking followed by no platform repeat activity
- repeated account behavior suggesting off-platform migration

## Enforcement
- soft warning
- hard warning
- temporary restriction
- payout hold
- penalty fee
- suspension
- permanent ban
